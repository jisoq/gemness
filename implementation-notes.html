Gemness Antigravity UX Implementation Notes

Last updated: 2026-05-20 KST. This file is maintained during implementation whenever the realized design differs from, clarifies, or extends the goal.

설계 결정

  • Default UXCodex main agent가 Gemness MCP 도구를 직접 오래 점유하지 않도록, 기본 가이던스는 antigravity reviewer subagent 위임으로 문서화했다. Gemness 서버 자체는 Codex subagent를 생성할 수 없으므로, 이 책임은 설치되는 AGENTS.md 트리거 블록과 gemness skill 지침에 둔다.
  • Final toolsask_antigravity, follow_up_antigravity, ask_antigravity_json, review_current_diff_with_antigravity는 blocking final-result 도구로 유지한다. 내부 구현은 기존 RunManager 기반 start -> await until terminal 흐름을 계속 사용해 heartbeat, cancel, idempotency, Observer 기록을 공유한다.
  • Detached APIstart_*, get_antigravity_run, await_antigravity_run, cancel_antigravity_run은 advanced background/batch API로 문서화했다. 기본 Codex flow에서 권장하지 않으며, bootstrap 설정에서는 start_*와 cancel_antigravity_run의 approval mode를 prompt로 바꾸었다.
  • Prompt eventsprompt.rendered는 canonical prompt 이벤트로 유지하고, prompt.sent에는 전체 prompt 대신 prompt_ref, redacted preview, 문자 수만 기록한다. 이렇게 하면 기존 raw 디버깅 표면과 review prompt 검증은 보존하면서 prompt 중복 저장을 줄일 수 있다.
  • Output events실제 agy stdout 원문은 Observer artifact 파일로 저장하고, antigravity.response 이벤트에는 preview, byte count, artifact reference, metadata만 남긴다. canonical MCP final result는 session.completed.result에 둔다. Public event view에서는 artifact의 로컬 파일 경로를 숨기고 name/bytes 같은 참조 정보만 노출한다.
  • JSON resultJSON 계열 MCP 응답에서는 raw_response 대신 response_preview를 반환한다. 구조화된 canonical 값은 data이고, 긴 원문은 Observer artifact/debug 표면으로 격리한다.
  • Noise cleanup최종 advisory text에서 파일 검색, background task 대기, "잠시 대기" 같은 진행 문구는 line-based heuristic으로 제거한다. 제거 후 본문이 비면 원문을 유지해 실제 답변 손실을 방지한다.

편차

  • Subagent runtimeGemness 서버 코드가 직접 Codex subagent를 spawn하도록 구현하지 않았다. MCP 서버는 Codex orchestration layer 밖에 있으므로, 실제 spawn 책임은 Codex skill/trigger 지침에 두는 것이 경계에 맞다.
  • Prompt file/stdin현재 실제 OS process argv에는 여전히 agy -p <prompt>가 사용된다. Observer 기록과 result metadata에서는 prompt argument를 [PROMPT_REDACTED]로 대체했지만, prompt file 또는 stdin 전달은 아직 구현하지 않았다. 이유는 현재 capability probe가 print flag 중심이고, stdin/file prompt 지원 여부를 안전하게 확인할 근거가 부족하기 때문이다.
  • Raw transcriptraw export에서는 canonical prompt.rendered가 전체 prompt를 담는다. 다만 같은 prompt를 prompt.sent에 다시 저장하지 않으므로, "긴 prompt 중복 저장 금지"의 핵심을 canonical-one-copy 방식으로 해석했다.
  • Model hint요청의 gpt 5.5-mini xhigh 표현은 설치 가이던스에 "가능하면 lightweight high-reasoning reviewer profile"로 넣었다. 현재 실행 환경의 사용 가능 모델명은 Codex host가 결정하므로, Gemness 패키지에서 강제하지 않는다.

트레이드오프

  • Wrapper 유지blocking final tools를 새 동기 실행 경로로 분리하는 대신 기존 detached worker를 내부적으로 재사용했다. 중복 구현은 줄고 cancel/heartbeat/idempotency는 보존되지만, 내부 명칭은 여전히 detached run 기반이다.
  • Raw stdout artifactraw stdout을 이벤트 payload에 그대로 넣는 방식보다 artifact reference가 main context와 API 응답을 깨끗하게 만든다. 대신 transcript 삭제 시 artifact 파일까지 같이 정리하는 관리 책임이 추가되어 delete path를 확장했다.
  • Heuristic filtering진행 문구 제거를 엄격한 parser 대신 보수적인 line filter로 구현했다. 빠르게 UX를 개선할 수 있지만, Antigravity 출력 문구가 바뀌면 추가 패턴이 필요할 수 있다.
  • Approval modeadvanced detached start/cancel을 prompt approval로 바꾸면 우발적 background 실행을 줄인다. 반대로 batch 자동화에서는 추가 승인이 필요할 수 있으므로, 필요하면 설치 옵션으로 조절할 여지가 있다.

미결 질문

  • Prompt transportagy가 prompt file 또는 stdin 기반 non-interactive 입력을 안정적으로 지원하는지 확인이 필요하다. 지원한다면 실제 process argv에서도 prompt를 제거하는 후속 패치가 가능하다.
  • Artifact APIObserver artifact reference만 충분한지, 브라우저에서 artifact를 열 수 있는 전용 로컬 endpoint가 필요한지 결정이 필요하다.
  • Filtering scope진행 문구 제거 heuristic이 한국어/영어 Antigravity 출력에서 충분히 안전한지 실제 장기 실행 transcript로 검증해야 한다.
  • Subagent policyCodex host에서 특정 reviewer model/effort를 지정할 수 없는 환경에서는 어떤 fallback 문구가 가장 적절한지 확인이 필요하다.

  • Observer검증 transcript는 .tmp/real-validation-transcripts에 저장되었고, Observer는 http://127.0.0.1:56755에서 해당 transcript를 표시하도록 현재 소스 기반 프로세스로 재기동했다.
  • Evidence실제 transcript에서 run.command.command_argv는 agy | -p | [PROMPT_REDACTED] 형태로 기록되었고, antigravity.response에는 response_preview, stdout_artifact, stdout_bytes, metadata, streaming만 남았다. response 및 stdout 필드는 없었다.
  • Result실제 최종 응답은 Gemness 스모크 테스트 연결이 성공적으로 확인되었습니다.였고, heartbeat 이벤트도 기록되었다.
  • Correction위 smoke 및 직접 diff review 시도는 Gemness 자체 검증에는 유효하지만, "main agent가 subagent를 spawn해서 맡기는 기본 UX" 검증으로는 부족했다. 직접 diff review 프로세스는 중단했고, 이후 parent/main agent는 Gemness 도구를 직접 호출하지 않고 antigravity reviewer subagent에게 review_current_diff_with_antigravity final-result 호출을 위임했다.
  • Subagent runSubagent nickname: Darwin. 지시사항: start/get/await/cancel detached API를 쓰지 말고 final-result review tool을 호출해 완료 결과 요약을 parent에게 반환할 것.
  • Accepted findingsDarwin의 advisory 중 진행문구 필터가 정상 조언을 지울 수 있다는 지적과 extensionless artifact 삭제 누락 가능성은 수용했다. 필터는 ellipsis가 있는 짧은 진행 라인 위주로 좁혔고, conversation/session 삭제는 run_id.* artifact 전체를 정리하도록 바꾸었다.
