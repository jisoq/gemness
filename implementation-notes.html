Gemness Antigravity CLI Implementation Notes

Antigravity CLI agy 전용 구현 기준을 기록한다. 이 파일은 현재 제품 동작을 설명하며 과거 backend 호환 지침을 보존하지 않는다.

정책

  • Gemness의 canonical backend는 Antigravity CLI agy이다.
  • 기본 실행 명령은 agy이며, Windows에서는 %LOCALAPPDATA%\agy\bin\agy.exe fallback을 확인한다.
  • Gemness는 Antigravity CLI의 비대화형 print mode(-p, --print, --prompt)만 사용한다.
  • Antigravity CLI 완료 출력을 캡처하고, Gemness가 MCP 응답 envelope와 Observer event를 합성한다. Windows에서는 CLI가 stdout/stderr 대신 콘솔에 직접 쓰는 경우가 있어 GEMNESS_AGY_CAPTURE_MODE=auto가 pywinpty 캡처를 사용한다.
  • 응답 metadata에는 run_id, conversation_id, command, cwd, duration_ms, exit_code, auth_status, streaming=false를 포함한다.
  • 모델 선택은 Antigravity CLI 설정 또는 /model에서 관리한다. Gemness는 model flag를 전달하지 않는다.

공식 문서 근거

  • 설치: https://antigravity.google/docs/cli-getting-started
  • 사용/설정: https://antigravity.google/docs/cli-using
  • 기능/플러그인: https://antigravity.google/docs/cli-features
  • 개요: https://antigravity.google/docs/cli-overview
  • CLI MCP config 위치: ~/.gemini/antigravity-cli/mcp_config.json 또는 .agents/mcp_config.json
  • remote MCP 설정은 serverUrl을 사용한다.

실물 확인

  • agy --version은 현재 환경에서 1.0.0을 반환했다.
  • agy --help는 -p, --print, --prompt, --continue, --conversation을 표시했다.
  • Gemness는 실제 Antigravity conversation ID를 감지하면 agy --conversation <id> -p <prompt>를 사용한다. ID가 없고 최신 대화 연속이 필요한 경우 agy --continue -p <prompt>를 사용할 수 있으며, branch/fallback에서는 짧은 summary prompt만 사용한다. 이전 프롬프트/응답/diff/file dump/log/transcript payload를 다시 전달하지 않는다.
  • 인증 상태는 health probe에서 best effort로 확인한다. 로그인 필요 문구가 stderr/stdout에 나타나면 auth_required로 보고한다.

검증 기준

  • runtime argv는 agy -p <prompt> 또는 help에서 확인된 동등 print flag 형태여야 한다.
  • Observer event는 antigravity.started, antigravity.response, antigravity.stderr, antigravity.exited를 사용한다.
  • 제품 문서, tests, bootstrap config에는 Antigravity CLI 설치와 agy --help/agy -p smoke test를 안내한다.
  • 잔여 브랜드 문자열은 공식 Antigravity config path와 모델 표시명에 한정한다.
